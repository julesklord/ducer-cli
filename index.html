System Design

HOW IT
WORKS

A passive Jump Hook intercepts music subcommands and delegates them to the plugin layer — no upstream code modified. Git merges stay clean.

ducer — system topology
User Prompt │ ▼ Gemini CLI Host ──── Jump Hook ────► Ducer Plugin Layer │ ┌───────────────────┼───────────────────┐ │ │ │ DucerCore Sensory Tools DawBridge (Orchestrator) (Audio/Media) (I/O Layer) │ │ │ Local DB AudioAnalyzer ReaperBridgeClient (Action Registry) (Waveform/Spec) (Web API + File IPC)
Tier 1
WEB CONTROL API

Hits REAPER's native HTTP endpoint directly for action execution and status. 1-second timeout, fails fast. No file I/O overhead.

Fast path · 127.0.0.1:8080
Tier 2
FILE IPC FALLBACK

Writes commands to ducer_commands.txt, polls ducer_response.txt. 100ms polling, 5-second timeout.

Fallback · APPDATA/REAPER/Scripts
Capabilities

PRODUCER
INTELLIGENCE

Every feature built around one principle: verify before executing, never hallucinate action IDs into a live session.

⬡
DAW BRIDGE

Universal abstraction for DAW orchestration. Implements five methods. Adding a new DAW — Ableton, Logic, anything — means implementing the interface. DucerCore never knows what's behind it.

DawBridge Interface
◈
ANTI-HALLUCINATION

Every action ID is validated via ReverseNamedCommandLookup before touching REAPER. Hallucinated IDs trigger a semantic search against the local action registry. No ghost actions in your session.

Anti-Hallucination Loop
◎
AUDIO ANALYSIS

Waveform visualization, spectral analysis, and vocal transcription via Gemini's multimodal 1M-token context. Standard, Advanced (HTML report), and Lite modes. The analysis hits the bytes directly — not just the filename.

Multimodal Engine
▶
SERVICE MODE

Persistent IPC loop that watches the command file every 200ms. A Lua script in REAPER writes a command — Ducer processes it via the LLM — writes the response back. Full bidirectional DAW↔LLM communication without leaving your session.

Persistent IPC
◇
LUA EXECUTION

The model generates and runs Lua scripts on demand. Complex routing (stem bussing, FX chains, automation) gets handled as dynamic code, not hardcoded actions. 300+ specialized Ducer scripts included.

Dynamic Code Gen
☰
MACRO LEARNING

Teach Ducer your custom workflow macros and name-to-ID mappings with learn_workflow_macro. Stored in a local JSON registry. No round-trip to the API for actions you use daily.

Local Registry
Usage

COMMANDS

Three modes. Natural language throughout.

ducer analyze — advanced mode
$ ducer analyze --file "./stems/kick_bus.wav" --advanced [Ducer] Audio cargado: kick_bus.wav (2.14 MB, audio/wav) [Ducer] Modo: advanced · Enviando a Gemini 1.5 Pro... Análisis Espectral: → Pico fundamental: 60Hz con armónico secundario notable en 120Hz → Rango 200-400Hz: ligeramente congestionado, considerar 2-3dB cut en 280Hz → Transiente: ataque en 8ms, sustain aceptable, decay limpio ✅ Reporte HTML generado → reports/Ducer_Report_kick_bus_1775850219850.html
ducer do — natural language DAW control
$ ducer do --query "sidechain route Track 1 to Track 2, add compressor" [Ducer] DAW Control Mode active → REAPER 7.x @ 127.0.0.1:8080 [Ducer] Ejecutando tool: execute_reaper_action → Validando action ID... ✓ (Transport: confirmed) → Generando Lua: sidechain_route.lua → Ejecutando en REAPER... OK (Web API) Sidechain routing configurado. Track 1 → Track 2 con compressor en pre-fader.
ducer service — persistent IPC loop
$ ducer service [Ducer] Service Mode Active. Listening for REAPER commands... [Ducer] Watching: %APPDATA%\REAPER\Scripts\ducer_commands.txt ↳ Trigger from REAPER Lua: reaper.WriteFile("ducer_commands.txt", query) [Ducer Service] Incoming: "analyze current track and suggest EQ settings" → Response written to ducer_response.txt
